SkySync

SkySync is a web application built using python, with flask as framework, gunicorn, nginx, HTML, CSS (local css and bootstrap) and javascript. It falls on the OpenWeatherMap API to retrieve, extract and display weather data using web-scraping techniques. This ensures a scalable and efficient weather service.

Features

  • Utilises geolocation (device's location) to retrieve weather data
  • Get weather information and forecasts for any city via search
  • Real-time weather updates
  • Weather forecasts (3-hour & 5-days forecast) on page load
  • Responsive design for varied displays
  • Easy integration with external APIs
  • Scalable using Gunicorn and Nginx

Installation

Prerequisites

  • Python 3.8.10 or higher
  • Flask
  • Gunicorn
  • Nginx
  • Git

Clone the Repository

git clone https://github.com/gitloper-azara/SkySync.git
cd SkySync

Install Dependencies

cd src
pip install -r requirements.txt

Configuration

Configure these environment variables in a .env file (These need to be configured in a gunicorn service file for deployment to production).

FLASK_APP=run.py
FLASK_ENV=development
WEATHER_API_KEY=weather_api_key
WEATHER_API_HOST=0.0.0.0
WEATHER_API_PORT=5003
PYTHONPATH=/path/to/SkySync/src

Make sure to replace weather_api_key with an actual weather API key from OpenWeatherMap.

Test

Unittest was used for all testing purposes during the development of SkySync (web-scraping). Navigate to the root directory to run tests.

cd SkySync
python3 -m unittest discover test

Running the Application

Using Flask's Development Server

export FLASK_APP=run.py
export FLASK_ENV=development
export PYTHONPATH=/path/to/SkySync/src
flask run

Using Gunicorn

Refer to gunicorn.service and insert content after the nano command.

nano /etc/systemd/system/gunicorn.service
sudo systemctl daemon-reload
sudo systemctl start gunicorn
sudo systemctl enable gunicorn

Setting up Nginx

Refer to nginx.default and insert content after the nano command.

nano /etc/nginx/sites-available/default
sudo ln -s /etc/nginx/sites-available/default /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

Only restart nginx when nginx -t (checking nginx syntax) passes without errors!
